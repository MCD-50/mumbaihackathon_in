Rules

General

  • All code must be written during the event. You cannot use existing code that you have written and finish it during the event
  • You can apply as an individual or a team
  • There will be seprate categories for Students & Professionals, the prize pool will be split accordingly
  • Only 1 person per team/project need apply
  • Teams can consist of upto a maximum of 5 members
  • All participants must bring their own laptops, no computers will be provided
  • All teams retain full ownership of what they have created during the Hackathon

Schedule

  • There will be two sessions for coding, 9am-6pm on 26th March and 9am-1pm on 27th March
  • You CAN code between the two sessions when you're not at the venue
  • Coding ends at 1 P.M on 27th March, from there onwards participants will demo their work

Project

  • All code, design, art, music, SFX, and assets must be created during the duration of the Hackathon. You are however permitted to use any publicly available open source resource (APIs, images, data sets, etc.) in your project that are and are free to make plans, create wireframes, and brainstorm prior to the event
  • All projects submitted for consideration for a prize will be subject to a full code and asset review in order to determine whether the work originated during the hackathon or not
  • All projects must be uploaded to GitHub
  • If you’re new to Github, read this guide​

Submission

  • Before a project demo, we will take down your project details, final team size and the GitHub location of your code
  • Once your project is submitted, DO NOT make any changes. We will keep an eye on the commits made on GitHub
  • Your project is eligible for submission only if it's in working condition and meet minimmum quality standards (Eg. A static website will not be considered as an eligible submission)
The organizers reserve the right to make changes to the event program
